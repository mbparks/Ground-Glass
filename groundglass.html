PROPERTY PROBE tries each way of writing a setting and keeps the one the body reports back. EVENT TAP opens the log for ten seconds: press it, then turn a dial on the camera, and every record the body sends is written out.

RELEASE PROBE tries each way of asking an EOS body for an exposure and watches the event stream for a new file, so it reports which one your body honours rather than which one returned success. It will make real exposures.

BENCH TEST walks the whole path against the body and reports a row per step, so a failure has an address rather than a shrug. It does not fire the shutter.

Live view over PTP is a polled JPEG, so it will look like a slideshow next to the GLASS link. Use WIRE for the knobs and GLASS for the picture.

Canon CCAPI over the network. Activate CCAPI on the body once with Canon's tool, join the camera to your network, then read the address and port off the camera screen.

CCAPI describes itself: the camera hands back every endpoint it supports. This link only calls what the body has said exists, so nothing here is guesswork. API MAP shows you the list.

The camera sends no cross origin headers, so this page can always show the live view but may not be able to read replies. PROBE finds out which half works from here.

Blind control sends commands the camera will act on but whose replies cannot be read back. Settings shown in the console are then a guess, not a readback.

Vendored libraries

None. Every byte in this file is hand written. The scopes, the rings, the PTP codec, and the JPEG plumbing are all first party. Embedded art is inline SVG and canvas drawing, authored for this instrument and covered by the same GPL-3.0 license as the code.

Freeing the camera for the WIRE link

  • Windows: the MTP class driver claims the body. Replace the driver on that interface with WinUSB using Zadig, then reconnect.
  • macOS: the system PTPCamera agent claims it. Quit it, or set the camera to a mode the agent ignores, then reconnect.
  • Linux: the gvfs gphoto2 volume monitor claims it. Stop that service or add a udev rule granting your user access to the device, then reconnect.

If the camera shows Err 70

Err 70 is the body reporting an internal fault, usually after being sent something it could not make sense of. Switch the camera off, take the battery out for around thirty seconds, reseat the lens, and switch it back on. It normally clears. If it returns without this instrument attached, the camera needs service rather than software.

A version of this instrument wrote a focus point using a payload layout that had been inferred rather than confirmed, and that is the likely cause if you saw Err 70 while using touch to focus. That feature has been removed. This build cannot write any property whose layout it has not confirmed: the refusal sits at the single point where payloads are built.

Known limitations

  • WIRE is written against the published Canon EOS PTP extensions and has not yet been proven against an R6. Connect it, run BENCH TEST in the LINK panel, and treat every write as unverified until those rows are green.
  • PTP live view is polled, so it is a slideshow, not video. The GLASS link is the honest monitor.
  • AETHER control depends on whether the camera tolerates a simple request. If it does not, live view still works and the knobs do not.
  • Video record start and stop, focus stacking, and image download are not in this version.
  • Scopes read the preview, not the sensor. They tell you about the picture you are being shown.
